The Problem

MoE Routing is a Linear Bottleneck

// Standard MoE routing gate (PyTorch)
def route(token, expert_weights):
    # Dot product against ALL experts → O(N)
    logits = token @ expert_weights.T  # [2048] × [2048, 64]
    top_k = topk(softmax(logits), k=8)
    return top_k  # Checked ALL 64, selected 8

The Solution

Replace dot(token, all_experts) with Ray Tracing

🎯 Core Insight

Expert centroids live in a 3D geometric space. Instead of checking every expert, fire a ray from the token position and let the GPU's RT Cores find the closest experts in O(log N) via BVH tree traversal — the same way games find which objects a bullet hits.

💡 Key Properties

  • ✦ BVH built once at model load (static geometry)
  • ✦ Zero construction overhead per token
  • ✦ RT Cores are dedicated silicon — idle during normal inference
  • ✦ Works on any RTX GPU (2060 through 5090)

Innovation

Spectral Routing — Solving Polysemy with Optics

🔬 The Polysemy Problem

The word "bank" can mean a financial institution or a river bank. Standard routing sends both to the same expert.

💎 Optical Solution

Rays carry context as a "wavelength". At domain boundaries, Snell's Law refracts compatible contexts through, while incompatible contexts trigger Total Internal Reflection — bouncing the token to the correct semantic domain.

Discovery

Experts Specialize by Syntax, Not Topic

Analyzed 64 experts × 16 layers across OLMoE, Qwen, and DeepSeek

❌ The Myth

"Expert 1 handles math, Expert 2 handles code, Expert 3 handles science."

Best topic specialist: only 6.8% (vs 3.3% random). Experts do NOT specialize by topic.

✅ The Reality

Experts cluster by token type:

C0: Content words (63%)   C1: Mixed (62%+21%)
C2: Transition (42%+19%)   C3: Structure (30%+27%)

This syntactic clustering is universal across all 3 architectures tested, and predictable enough to map into a geometric BVH structure.
